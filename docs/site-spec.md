# MORI CROWN! サイト構築仕様

最終更新: 2026-02-14

## 1. 目的

- MORI CROWN! の世界観と所属タレントを分かりやすく伝える
- ニュース確認、配信視聴、グッズ購入、応募を1サイトで完結させる
- 初見ユーザーと既存ファンの両方が迷わず回遊できる情報設計にする
- ファンが二次創作と応援活動をしやすい導線を用意する

## 2. 想定ユーザーと主行動

| ユーザー | 主な目的 | 最短導線 |
| --- | --- | --- |
| 初見ユーザー | どんな事務所か知る | `ホーム` -> `タレント一覧` -> `タレント詳細` |
| 既存ファン | 最新情報を追う | `ホーム` -> `ニュース一覧` -> `ニュース詳細` |
| 配信視聴ユーザー | すぐ配信を見る | `ホーム` -> `配信スケジュール` |
| 購入意欲ユーザー | グッズを買う | `ホーム` or `ニュース詳細` -> `ショップ` |
| 応募希望者 | 募集要項を確認して応募 | `ホーム` -> `オーディション` -> `応募フォーム` |
| 創作ファン | 二次創作の条件確認と素材取得 | `タレント詳細` -> `ファン活動ガイド` |

## 3. サイトマップ（URL設計）

| ページ名 | URL | 役割 |

## 4. グローバルナビゲーション仕様

### 4.1 ヘッダー主メニュー（優先6項目）

| メニュー名 | リンク先 | 内容 |
| --- | --- | --- |
| ホーム | `/` | サイト全体の入口 |
| ニュース | `/news` | 配信・イベント・グッズ告知 |
| タレント | `/talents` | 所属VTuber一覧と詳細導線 |
| 配信 | `/schedule` | 直近配信の確認 |
| ショップ | `/shop` | グッズ購入導線 |
| オーディション | `/audition` | 募集情報と応募導線 |

### 4.2 ユーティリティ導線

| 項目 | リンク先 | 配置 |
| --- | --- | --- |
| 会社概要 | `/company` | ヘッダー右端（PC）とフッター |
| お問い合わせ | `/contact` | ヘッダー右端（PC）とフッター |

補足:

- モバイルはハンバーガーメニュー化し、表示順は主メニュー優先
- 現在ページは `is-active` で強調表示する
- ヘッダーに `今配信中` または `次回配信` の1クリック導線を設置する

## 5. ページ別コンテンツ要件

### 5.1 ホーム（`/`）

- ヒーロー: 事務所コンセプト、主要CTA（タレント、ニュース）
- ライブ導線: `今配信中` または `次回配信`
- ピックアップニュース: 新着3-4件
- タレント紹介: 主要メンバーカード
- ショップ導線: 季節グッズ訴求
- オーディション導線: 募集中の訴求

### 5.2 ニュース一覧（`/news`）

- カテゴリフィルタ: `すべて / 配信 / イベント / グッズ / 重要`
- 記事カード: 日付、カテゴリ、タイトル、サムネイル、詳細リンク
- ページャーまたは「もっと見る」

### 5.3 ニュース詳細（`/news/{slug}`）

- 本文、関連画像、関連タレントリンク
- 前後記事リンク
- 関連ニュース3件
- CTA: `配信を見る` `ショップを見る` `タレント詳細を見る`

### 5.4 タレント一覧（`/talents`）

- 期生または属性タグのフィルタ
- 各カードから `タレント詳細` へ遷移

### 5.5 タレント詳細（`/talents/{slug}`）

- 基本プロフィール
- キャラクター紹介
- 代表配信カテゴリ
- 最新配信リンク
- 関連ニュース
- グッズ導線
- ファン活動ガイド導線（タグ・二次創作ルール・素材配布）

### 5.6 配信スケジュール（`/schedule`）

- 週間カレンダー表示
- タレント別絞り込み
- 各枠から配信URLへ遷移
- `いま視聴する` ボタンを最上段に配置

### 5.7 ショップ（`/shop`）

- 注目グッズ
- 販売期間
- 購入導線（外部ECの場合は遷移前に案内表示）

### 5.8 オーディション（`/audition`）

- 応募対象、歓迎条件、NG事項
- 選考フロー（応募 -> 一次 -> 二次 -> 合否）
- 応募フォーム導線

### 5.9 ファン活動ガイド（`/fan-works`）

- 二次創作ガイドライン（可/不可事項）
- ハッシュタグ規定（総合、FA、切り抜き）
- 配布素材（ロゴ、サムネフレーム、カラーパレット）
- 問い合わせ導線（利用確認が必要なケース）

### 5.10 会社概要（`/company`）

- 事務所紹介
- ミッション/ビジョン
- 主要実績（架空実績）
- ガイドライン要約（詳細は `/fan-works` へ誘導）

### 5.11 お問い合わせ（`/contact`）

- 問い合わせ種別（法人、メディア、一般）
- 必須項目入力フォーム
- 送信後 `/contact/thanks` へ遷移

## 6. 主要遷移フロー

### 6.1 初見ユーザー導線

`/` -> `/talents` -> `/talents/{slug}` -> `/schedule` -> `/shop`

### 6.2 情報追従ユーザー導線

`/` -> `/news` -> `/news/{slug}` -> `/talents/{slug}` -> `/schedule`

### 6.3 応募ユーザー導線

`/` -> `/audition` -> `応募フォーム` -> `送信完了`

### 6.4 創作ファン導線

`/talents/{slug}` -> `/fan-works` -> `素材ダウンロード` -> `タグ投稿`

### 6.5 問い合わせユーザー導線

`/company` -> `/contact` -> `/contact/thanks`

## 7. UX・見やすさ基準

- 文字サイズは本文 `16px` 以上を基準にする
- 通常テキストのコントラスト比は `4.5:1` 以上を満たす
- キーボード操作のみで全リンクとフォームに到達可能にする
- フォーカス表示を必須にする（`:focus-visible`）
- 画像には用途に応じた代替テキストを付与する
- 動きのある演出は `prefers-reduced-motion` に対応する

## 8. ナビゲーション運用ルール

- ヘッダーはスクロール時も固定表示
- 下層ページはパンくずを表示する
- 詳細ページには「一覧へ戻る」リンクを置く
- 主要CTAボタンは各ページ1-2個に制限し、選択を迷わせない

## 9. MVP実装範囲

- 第2段階（拡張）:
  - 検索機能、タグ横断一覧、会員向けページ、CMS連携
- データ管理:
